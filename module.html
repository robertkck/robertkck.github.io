module.processing.compute_coinc(path, startyear, endyear, unit)[source]¶

Computes a co-occurence matrix of IPC codes

This function loads the patent count matrices and generates co-occurence matrices.

Args:
path (str): Specify data directory startyear (int): Between 1990 and 2014 endyear (int): Between 1991 and 2015 (inclusive) unit (str): Specifies the unit of observation to load. Either ‘ctry’, ‘appln’ or ‘pers’
Returns:
Pandas dataframe with IPC codes as columns and rows. The values are the absolute frequencies of co-occurence on the level of the specified unit
module.processing.country_dict(ctry, *to)[source]¶

Country code dictionary

This function tries to facilitate the conversion between 2-digit and 3-digit country codes as well as country names. Moreover, EU membership can be loaded.

Args:
ctry (pd.Series): Pandas series including 2-digit country codes to (str): Select conversion to ‘iso2c’, ‘country.name’ or ‘eu’
Returns:
Pandas series with the new country codes, names or EU membership

module.processing.load_patents(path, startyear, endyear, unit='ctry')[source]¶

Loads aggregated patent data

This function loads patent counts aggregated at the country, inventor or application level. The data is formatted as a count matrix with all >640 IPC codes as columns. Fractional counts are implemented for multiple technology codes and for the country level data.

Args:
path (str): Specify data directory startyear (int): Between 1990 and 2014 endyear (int): Between 1991 and 2015 (inclusive) unit (str): Specifies the unit of observation to load. Either ‘ctry’, ‘appln’, ‘appln_eu’ or ‘pers’
Returns:
Pandas dataframe with the specified observation level. Returns a matrix of patent counts by IPC code.
